h5基础（5.13）

h5说明

  • 狭义：Html5====html4 升级版;html5--> 新标签（语义标签）
  • 广义：h5属于综合的开发应用平台(H5+C3+Js) 网页，游戏，app....
  • H5语法结构:
    • < !doctype html> 指定网页文档类型
    • 严格模式 （strict.dtd）：区分标签大小写 html:4s + tab 或则 html:xs
    • 松散模式 （loose.dtd）html:4t
    • 过渡模式 （transitional.dtd）html:xt
  • 注意：网页中doctype不能去掉;怪异模式：不再按照w3c规范渲染，使用浏览器自己内部的渲染引擎去渲染

c3基础（5.14）

简介

  • 在CSS2基础上，增强或新增了许多特性， 弥补了CSS2的众多不足之处，使得Web开发变得更为高效和便捷。
  • CSS3的现状
    • 1、浏览器支持程度差，需要添加私有前缀
    • 2、移动端支持优于PC端
    • 3、不断改进中4、应用相对广泛
  • 如何对待
    • 1.坚持渐进增强原则
    • 2.考虑用户群体
    • 3.遵照产品的方案

手册介绍

  • [ ] 可选项;| 多选一;{ } 范围;|| 可以选择多个;* 出现0 次或者多次;? 出现0次或者1次

选择器

  • 属性选择器
    • [属性] { 属性: 值; 属性: 值;}
    • [class]：将页面中所有具有class属性的标签选中
    • [class="one"]：将class属性等于one的标签选中
    • [class^="o"]：将class属性的值以o开始的标签选中
    • [class$="o"]:将class属性的值以o结束的标签选中
    • [class*="o"]:将class属性的值中包含o的标签选中
  • 伪类选择器
    • 结构伪类html标签结构必须是嵌套结构（子元素和父元素）
      • li:first-child{}在结构伪类中标签first-child 选中的是父元素中第一个对应的子元素
      • :last-child 将li在父元素中最后一个选中
      • :nth-child(1) 选中第n个元素
      • :nth-last-child(2) 选择倒数n个标签
      • :nth-child(odd) 选择奇数位置的标签
      • :nth-child(even) 选中偶数位置的标签
      • 注意：n的取值方式：具体值 从0 开始;表达式 an+b
    • 空伪类 div：empty {}将页面中div标签没有内容选中
    • 排除伪类(not);:not(选择器) {}
    • 目标伪类(target) 目标伪类必须和锚链接配合使用
  • 伪元素选择器
    • ::first-letter 作用选中第一个字母（汉字）
    • ::first-line 选中第一行文字（字母）
    • ::selection 标签选中状态下的样式 只能设置与颜色相关的属性
    • ::before ::after（伪元素： 不是一个真正的元素）
      • 伪元素必须配合content使用；伪元素属于行内元素
      • 推荐使用单冒号，浏览器会自动添加补充（兼容）；因为在ie低版本中不识别这两个伪元素两个冒号，所以这两个伪元素使用一个冒号

c3新样式

  • 颜色表示方式 HSL:H: 色调 0-360;S: 饱和度 0%-100%;L: 亮度 0%-100%
  • 半透明效果:rgba() a： 0 -1;hsla() ===> 半透明效果 a: 0-1;
  • 文本修饰：文字阴影(可以加多个)

  • 盒子模型

c3背景，线性渐变，径向渐变，2D,3D(5.16)

背景

  • 背景尺寸 background-size:对背景图片进行裁切
    • 设置具体值长度或则百分比
    • 100% auto 或者 auto 100%
    • cover 背景图片始终会填充满父容器（按照自己的伸缩比调整大小）
    • contain 背景图片会完整显示在父容器中
  • 背景参考原点 background-origin:padding-box(默认边距为原点)||content-box（内容）||border-box（边框）
  • 背景裁切(将背景图片放在内容区域显示)background-clip:content-box||padding-box||border-box（默认值）;
  • 多背景 background:url()no-repeat top left,url()no-repeat bottom right;
    • 一个容器可以同时设置多张背景图片
    • 如果一个容器需要设置多背景，那么背景颜色不能设置到gackground复合属性中，要在后面单独设置

渐变

  • 线性渐变
    • 1.开始颜色2.结束颜色3.渐变的距离（范围）4.渐变的方向
    • background-image:linear-gradient(to right,red,green)
    • 方向:to right|left|top|bottom;
    • 使用角度：0deg渐变方向是从下向上；90deg渐变方向是从左到右，使用度数时按照顺时针
    • background-image:linear-gradient(to right,red 10%,green %10,green %20,yellow 20%...)可以保证每个色段一样
    • 渐变颜色后的百分比是相对父容器的宽度（没有设置background-size），否则百分比是以background-size的值为参数设置的
    • 颜色可以是上一个渐变颜色的结束，也可以是下一个渐变颜色的开始
  • 径向渐变
    • 1.开始颜色2.结束颜色3.圆心位置4.渐变的范围（圆的半径）
    • background-image:radial-gradient(100px(渐变的范围-半径)at 20px 50px (圆心位置),red,blue)
    • 圆心位置设置 right|left|top|bottom
    • 设置一个渐变椭圆，增加一个半径，第一个为X半径，第二个为Y半径

过度动画

  • 帧动画：动画按照像素去执行
  • 补间动画：1.动画的开始状态2.动画的结束状态3.动画的执行时间
    • transition-property:all;所有的属性发生动画的效果（可以设置单个属性如：width）
    • transition-duration:1s;设置动画执行时间
    • transition-delay:1s;设置动画延时时间
    • transition-timing-function:linear|ease| ease-in|ease-out|ease-in-out;设置动画执行的类型（速度）
    • 复合属性：transition:all 2s 2s linear

2D转换

  • 位移 transform:translate(x,y)
    • 可以设置一个百分比，百分比相对元素自己
    • x,y可以设置负值，代表方向
  • 旋转 transform:rotate(-30deg);旋转中可以设置角度（可以取正值和负值）
  • 缩放 transform scale(0.5);值>1的时候放大，值<1的时候缩小，不能取负值,可以设置两个值，一个X一个Y
  • 旋转原点 transform-origin:20px,100px;
    • 使用关键字 left top right bottom center
    • 通过设置具体值
    • 默认元素的旋转原点在中心位置 center
  • 倾斜 skew(30deg,20deg);

3D转换

  • c3中的坐标系：X:从左到右Y:从上到下Z:指向我们自己
  • 透视：辅助识别3D效果，不是一个必须的属性；两种设置方式
    • perspective:1000px; 以单独的属性给父元素设置
    • transform:perspective(1000px) translateZ(100px);给trnasform属性的值设置
    • perspective的取值不一定（600-1000）
  • 把当前元素转换成3D transform-style: preserve-3d

动画和伸缩布局及多列布局

动画 特点：不需要任何触发条件就可以执行

  • 动画的定义： @keyframes move{from{动画的开始状态} to{动画的结束状态}};
  • 其他设置方式： @keyframes move{0%{动画的开始状态} ..多个..100%{动画的结束状态};百分比是以动画执行时间为参照的
  • 属性连写调用 animation: move 2s 3s(如果有延迟执行时间，那么它必须在执行时间后面) linear infinite;
  • 属性详解
    • 调用动画 animation-name:move;
    • 动画执行时间 animation-duration:2s;
    • 动画执行次数 animation-iteration-count:infinite;
    • 设置动画延时执行 animation-delay:2s;
    • 设置动画的类型：速度 animation-timing-function:linear
    • 设置拟波动画（动画执行完后反向执行）animation-direction:alternate
    • 设置动画结束时的状态 animation-fill-mode:forwards
    • 动画结束时的状态会受到执行次数的影响
  • steps()步长，设置一个运动多少次走完，有步长不能加匀速
  • animation-play-state:paused;动画暂停

伸缩布局

  • display:flex;给子元素的直接父元素设置为伸缩盒子
  • 伸缩盒子特点
    • 只要一个容器设置为了伸缩盒子，那么该容器就具备了主轴和侧轴
    • 主轴：默认是从左向右显示
    • 侧轴：始终垂直于主轴，从上向下
    • 伸缩盒子中的元素是按照主轴方向对齐显示
  • 调整子元素在主轴方向的对齐方式 justify-content:flex-start（左对齐）|flex-end（右对齐）|center（居中）|space-around（环绕）|space-between（两端对齐）
  • 调整子元素在侧轴的对齐方式 align-items:flex-start|flex-end|center|stretch(默认拉伸)
  • 调整主轴方向 flex-direction:row（默认）|row-reverse（反转）|column（竖）|column-reverse（从下到上）
  • 设置盒子是否换行显示 flex-wrap:nowrap(默认不换行)|wrap（换行）
  • 设置盒子换行后的对齐方式 align-content:flex-start|flex-end|center|space-between|space-around|stretch;(换行后的盒子才能生效)
  • 设置伸缩盒子中子元素的属性
    • flex:1;设置子元素的伸缩比
    • order:1;设置子元素的显示位置（顺序，值越大越靠后）
    • align-self:flex-start|flex-end|center;设置子元素自己的对齐方式

补充知识：绝对定位的盒子居中，可以用2D3D移动元素的方式让元素居中对齐，设置百分比，因为它移动的是自身的百分比；与margin-left:-10px对比，优势在于当前居中盒子的宽度可变

地理定位

  • 在HTML规范中，增加了获取用户地理信息的API，这样使得我们可以基于用户位置开发互联网应用，即基于位置服务 (Location Base Service)。
  • 过IP地址、三维坐标、GPS、Wi-Fi、手机信息等多种方式获取地址信息
  • 隐私:HTML5 Geolocation 规范提供了一套保护用户隐私的机制。必须先得到用户明确许可，才能获取用户的位置信息。
  • 获取当前地理信息 navigator.geolocation.getCurrentPosition(successCallback, errorCallback)
  • 获取实时地理信息 navigator.geolocation.watchPosition(successCallback, errorCallback)
  • 当成功获取地理信息后，会调用succssCallback，并返回一个包含位置信息的对象。获取地理信息失败后，会调用errorCallback，并返回一个包含错误信息的对象。
  • position.coords.latitude纬度 position.coords.longitude经度
  • position.coords.accuracy精度 position.coords.altitude海拔高度
  • 应用:在现实开发中，通过调用第三方API（如百度地图）来实现地理定位信息，这些API都是基于用户当前位置的，并将用位置位置（经/纬度）当做参数传递，就可以实现相应的功能。

    • clear() 清空所有存储内容
    • key(n) 以索引值来获取存储内容
    • $("form").serializeArray() 可以把表单里的数据转换成一个对象数组

分辨率

  • 平常所说的屏幕尺寸都是指对角线，单位-英寸
  • 逻辑分辨率:屏幕的宽和高（px）
  • 设备分辨率:屏幕里面一共拥有的物理像素点(图像显示的最小单位)的个数
  • 失真
    • 在iphone4 普通图片被放大了

    • srcset="./images/02.png 2x,./images/3.png 3x "
    • 工作里面根据实际情况去采取措施

视口

  • 视口 html标签的宽*高
  • 布局视口-被手机厂商设置宽度为980px的视口--使用绝对长度单位写的元素被缩放
  • 理想视口-视口宽度=屏幕的宽度 使用绝对长度写的元素没有被缩放
    • <meta name="viewport" content="width=device-width,initial-scale=1,user-scalable=no,maximum-scale=1,minimum-scale=1" >
    • name: 告诉浏览器 要对当前的页面做哪些配置
    • content:要对视口的哪些内容进行设置
    • initial-scale：页面初始化的时候放大的倍数
    • user-scalable:是否允许用户缩放页面 yes or no
    • maxmum-scale:如果允许放大的话 可以放大的倍数
    • minimum-scale:如果允许放大的话 可以放大的倍数

触屏事件(鼠标事件不能够满足多指触控的事件需求)

rem+媒体查询移动端布局 (5.31)

  • 流式布局（百分比布局）问题：牵一发动全身，层层依赖太强了
  • rem：相对长度单位-相对与根元素 html-fontsize
  • 基础值：媒体查询（和设计稿等宽）里面 定义的html的标签的fontsize的大小
  • 为什么要写fontsize=100px 为了计算方便 并且 谷歌浏览器最小字体是12px （1rem=1px 是没有效果的）
  • 公式 设计稿的宽度（640）/要适配的屏幕的宽度=基础值/要适配的屏幕的html的fontsize
  • 把媒体查询换成js来实现 但是不常用 css能实现的功能最好不用js来实现
  • 在pc端 滚动条是占宽度的 移动端的滚动条是不占宽度的
  • 实现适配不同屏幕的代码的写法步骤：1 使用rem单位还原设计图； 2 媒体查询+rem去做适配
  • px:绝对长度单位 在代码里面使用了绝对长度单位 这个元素它的大小是固定的（不会随着屏幕的变化而发生变化）
  • em:相对长度单位（） 相对于自己的字体大小-font-size 谷歌浏览器默认最小字体是 12px 谷歌浏览器默认字体是16px
  • px转换为媒体查询的rem（转化前代码：@media screen and(width:640px){ html {font-size: 100px; }}）
